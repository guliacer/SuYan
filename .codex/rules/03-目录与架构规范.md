# 目录与架构规范

## 适用范围
适用于项目目录树、模块边界、路径别名、Electron 进程隔离和文件归属。

## 规则条目

### R1: 项目目录必须采用固定骨架
**断言**：项目源码必须按以下目录组织；新增一级源码目录前必须先更新本规则文件。
**示例**：
```text
W:\提示词\
  electron\
    main\
    preload\
    shared\
  src\
    app\
    components\
    features\
      library\
        components\
        hooks\
        store\
        types\
        utils\
    styles\
    types\
  tests\
    unit\
```
**违规特征**：在项目根目录散落 `main.ts`、`preload.ts`、`LibraryPanel.tsx`，或创建未定义的 `backend/`、`server/`、`database/` 目录。

### R2: Electron 三层代码必须物理隔离
**断言**：主进程代码只能位于 `W:\提示词\electron\main\`；preload 代码只能位于 `W:\提示词\electron\preload\`；React 渲染层只能位于 `W:\提示词\src\`。
**示例**：
```text
W:\提示词\electron\main\clipboard\readClipboardImage.ts
W:\提示词\electron\preload\suyanApi.ts
W:\提示词\src\features\library\components\LibraryView.tsx
```
**违规特征**：`src/` 内出现 `ipcMain`、`BrowserWindow`、`node:fs`；`electron/main/` 内出现 React 组件。

### R3: 路径别名只能指向渲染层 src
**断言**：`@/` 只能映射到 `W:\提示词\src\`；渲染层不得通过别名导入 `electron/main` 或 `electron/preload`。
**示例**：
```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
}
```
**违规特征**：配置 `"@electron/*": ["electron/*"]` 后在 React 组件中导入 Electron 主进程模块。

### R4: library 功能必须封装在 feature 模块内
**断言**：素材库相关组件、hooks、store、types、utils 必须放在 `W:\提示词\src\features\library\` 下；跨功能通用组件才允许放入 `W:\提示词\src\components\`。
**示例**：
```ts
// W:\提示词\src\features\library\index.ts
export { LibraryView } from "./components/LibraryView";
export { useLibraryStore } from "./store/useLibraryStore";
export type { LibraryItem } from "./types/library";
```
**违规特征**：把 `PromptEditor.tsx` 放到 `src/components/`，或在 `src/app/` 中直接堆叠素材库业务逻辑。

### R5: 业务类型必须集中声明并复用
**断言**：`LibraryItem`、`LibraryFile`、`ImportResult`、`ExportResult` 等业务类型必须集中在 `W:\提示词\src\features\library\types\` 或 `W:\提示词\src\types\`，不得在多个组件中重复定义。
**示例**：
```ts
export type LibraryItem = {
  id: string;
  title: string;
  imageFileName: string;
  prompt: string;
  negativePrompt: string;
  tags: string[];
  createdAt: string;
  updatedAt: string;
};
```
**违规特征**：`LibraryList.tsx`、`PromptEditor.tsx`、`useLibraryStore.ts` 各自定义结构不一致的 `Item` 类型。
