# 数据库与状态管理规范

## 适用范围
适用于本地素材库文件、图片存储、数据 schema、Zustand store、持久化写入和数据迁移。

## 规则条目

### R1: 第一版只能使用 library.json 作为元数据存储
**断言**：素材库元数据必须保存为应用数据目录内的 `library.json`；第一版不得使用 SQLite、IndexedDB、LocalStorage、远程数据库或浏览器缓存作为主存储。
**示例**：
```json
{
  "schemaVersion": 1,
  "items": []
}
```
**违规特征**：新增 `db.sqlite`、`indexedDB.open("library")`、`localStorage.setItem("library", ...)` 或远程保存素材数据。

### R2: library.json schema 必须固定
**断言**：`library.json` 的顶层字段必须为 `schemaVersion`、`items`、`updatedAt`；每个 item 必须包含 `id`、`title`、`imageFileName`、`prompt`、`negativePrompt`、`tags`、`createdAt`、`updatedAt`。
**示例**：
```ts
export type LibraryFile = {
  schemaVersion: 1;
  updatedAt: string;
  items: LibraryItem[];
};

export type LibraryItem = {
  id: string;
  title: string;
  imageFileName: string;
  prompt: string;
  negativePrompt: string;
  tags: string[];
  createdAt: string;
  updatedAt: string;
};
```
**违规特征**：某些 item 使用 `imagePath`，另一些使用 `file`；或缺失 `schemaVersion` 导致无法迁移。

### R3: 托管图片与外链索引必须严格分开
**断言**：默认 managed 导入必须复制到应用数据目录的 `images/` 下，文件名必须使用 item `id` 加原始扩展名或安全推断扩展名。允许 `mediaStorage: { kind: "external", rootId, relativePath }` 记录用户目录中的外链素材；外链条目只能保存相对路径，绝不把原始像素复制进 `images/`。
**示例**：
```text
%APPDATA%\SuYan\library.json
%APPDATA%\SuYan\images\018f4cc0-4fd2-7a0e-8e7a-63b7f99f0b12.png
```
**违规特征**：把图片转为 base64 写进 `library.json`；external 保存未注册 root 的绝对路径；扫描外链目录时调用 `copyFile` 或 `writeImportMediaFile`。

### R3.1: external 删除与压缩铁律
**断言**：删除 external 条目只能删除 `library.json` 索引和应用自有 `thumbnails/` 缓存，绝不 `unlink` 用户原文件。压缩、转码、格式替换只允许 managed 条目；external 默认跳过，禁止写回原文件。

**违规特征**：external 删除路径落到挂载根；批量压缩/视频转码将 external 文件作为输出目标。

### R4: Zustand store 不得直接持久化文件
**断言**：`useLibraryStore` 只能管理内存状态和调用 `window.suyanApi`；不得在 store 中直接访问文件系统、LocalStorage 或 Electron API。
**示例**：
```ts
export const useLibraryStore = create<LibraryState>((set, get) => ({
  items: [],
  selectedItemId: null,
  load: async () => {
    const result = await window.suyanApi.readLibrary();
    if (result.ok) set({ items: result.data.items });
  },
  save: async () => {
    const library = buildLibraryFile(get().items);
    return window.suyanApi.saveLibrary(library);
  },
}));
```
**违规特征**：store 中出现 `localStorage.setItem`、`fs.writeFile`、`ipcRenderer.invoke` 或保存 base64 图片。

### R5: schema 变化必须提供迁移函数
**断言**：任何修改 `LibraryFile` 或 `LibraryItem` 字段的提交，必须同步新增 `W:\提示词\src\features\library\utils\migrateLibrary.ts` 中的迁移逻辑和 Vitest 单元测试。
**示例**：
```ts
export function migrateLibrary(input: unknown): LibraryFile {
  const file = parseLibraryFile(input);
  if (file.schemaVersion === 1) {
    return file;
  }
  throw new Error("Unsupported library schemaVersion");
}
```
**违规特征**：直接把 `schemaVersion` 从 `1` 改成 `2`，但没有旧数据迁移和测试。
