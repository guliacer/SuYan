# 通信与数据获取规范

## 适用范围
适用于 Electron IPC、preload 暴露 API、系统剪切板、文件导入导出、图片复制、ZIP 分享包和数据读写调用链。

## 规则条目

### R1: 渲染层只能访问 window.suyanApi
**断言**：`W:\提示词\src\` 下所有 React 代码只能通过 `window.suyanApi` 调用桌面能力；不得直接导入 `electron`、`node:fs`、`node:path` 或 `ipcRenderer`。
**示例**：
```ts
declare global {
  interface Window {
    suyanApi: {
      readLibrary: () => Promise<LibraryFile>;
      saveLibrary: (library: LibraryFile) => Promise<SaveResult>;
      importImageFiles: () => Promise<ImportImagesResult>;
      importClipboardImage: () => Promise<ImportClipboardImageResult>;
      exportZip: (itemIds: string[]) => Promise<ExportZipResult>;
      importZip: () => Promise<ImportZipResult>;
    };
  }
}
```
**违规特征**：React 组件中出现 `ipcRenderer.invoke(...)`、`fs.readFile(...)` 或 `clipboard.readImage()`。

### R2: preload 只能暴露最小 API
**断言**：`W:\提示词\electron\preload\suyanApi.ts` 只能暴露经过命名的业务方法；不得暴露通用 `send`、`invoke`、`on` 或原始 `ipcRenderer`。
**示例**：
```ts
contextBridge.exposeInMainWorld("suyanApi", {
  importClipboardImage: () => ipcRenderer.invoke(ipcChannels.imageImportFromClipboard),
  readLibrary: () => ipcRenderer.invoke(ipcChannels.libraryRead),
});
```
**违规特征**：`contextBridge.exposeInMainWorld("electron", { ipcRenderer })` 或暴露 `invoke(channel, payload)`。

### R3: IPC 参数和返回值必须结构化
**断言**：所有 IPC 返回值必须是 `{ ok: true, data }` 或 `{ ok: false, error }` 结构；不得让渲染层依赖抛出的原始异常。
**示例**：
```ts
export type IpcResult<T> =
  | { ok: true; data: T }
  | { ok: false; error: { code: string; message: string } };
```
**违规特征**：`throw new Error("failed")` 后直接让组件捕获并显示 `error.message`，或返回 `null` 表示多种失败原因。

### R4: 剪切板图片导入必须由主进程完成
**断言**：系统剪切板图片读取、PNG/JPEG 编码、图片落盘必须位于 `W:\提示词\electron\main\clipboard\`；渲染层只接收导入结果。
**示例**：
```ts
export type ImportClipboardImageResult = IpcResult<{
  itemId: string;
  imageFileName: string;
  width: number;
  height: number;
}>;
```
**违规特征**：在 React 组件中调用浏览器 Clipboard API 读取系统图片，或把 base64 大图长期存入 Zustand。

### R5: ZIP 导入导出格式必须稳定
**断言**：导出的 ZIP 根目录必须包含 `data.json` 和 `images/`；`data.json` 必须包含 `schemaVersion` 和 `items`；导入器必须拒绝缺少 `schemaVersion` 的包。
**示例**：
```text
share.zip
  data.json
  images/
    018f4cc0-4fd2-7a0e-8e7a-63b7f99f0b12.png
```
```json
{
  "schemaVersion": 1,
  "items": [
    {
      "id": "018f4cc0-4fd2-7a0e-8e7a-63b7f99f0b12",
      "imageFileName": "018f4cc0-4fd2-7a0e-8e7a-63b7f99f0b12.png",
      "prompt": "柔和自然光下的陶瓷杯"
    }
  ]
}
```
**违规特征**：导出只有图片没有 `data.json`，或导入时默默接受未知结构并写入素材库。
