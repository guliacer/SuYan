# 核心铁律与避坑

## 适用范围
适用于安全边界、性能红线、数据可靠性、依赖控制和第一版范围控制。

## 规则条目

### R1: 渲染进程不得获得 Node.js 能力
**断言**：创建 `BrowserWindow` 时必须设置 `nodeIntegration: false`、`contextIsolation: true`、`sandbox: true`；渲染进程不得出现 `node:fs`、`electron`、`ipcRenderer` 直接导入。
**示例**：
```ts
const mainWindow = new BrowserWindow({
  webPreferences: {
    preload: preloadPath,
    nodeIntegration: false,
    contextIsolation: true,
    sandbox: true,
  },
});
```
**违规特征**：`webPreferences.nodeIntegration = true`，或 React 组件中写 `import { ipcRenderer } from "electron"`。

### R2: IPC channel 必须白名单化
**断言**：所有 IPC channel 必须集中声明在 `W:\提示词\electron\shared\ipcChannels.ts`；不得在组件中拼接字符串 channel。
**示例**：
```ts
export const ipcChannels = {
  libraryRead: "library:read",
  librarySave: "library:save",
  imageImportFromClipboard: "image:import-from-clipboard",
  archiveExportZip: "archive:export-zip",
  archiveImportZip: "archive:import-zip",
} as const;
```
**违规特征**：`ipcRenderer.invoke("image:" + action)` 或在多个文件中散落 `"library:save"` 字符串。

### R3: 第一版不得引入重型基础设施
**断言**：第一版 `package.json` 不得包含数据库、ORM、远程后端、登录鉴权或云存储 SDK；素材库只能使用本地 JSON 与图片目录。
**示例**：
```json
{
  "forbiddenV1Dependencies": [
    "prisma",
    "sqlite3",
    "better-sqlite3",
    "firebase",
    "@supabase/supabase-js",
    "next",
    "express"
  ]
}
```
**违规特征**：为本地素材管理引入 Prisma、SQLite、Supabase、Express、Next.js API Routes。

### R4: 导入图片必须复制到应用数据目录
**断言**：`library.json` 中只能保存应用数据目录内的 `imageFileName` 或相对路径；不得把用户原始外部路径作为唯一图片引用。
**示例**：
```json
{
  "id": "018f4cc0-4fd2-7a0e-8e7a-63b7f99f0b12",
  "imageFileName": "018f4cc0-4fd2-7a0e-8e7a-63b7f99f0b12.png",
  "prompt": "一只白色陶瓷杯，柔和自然光"
}
```
**违规特征**：只保存 `"imagePath": "C:\\Users\\Alice\\Desktop\\a.png"`，导致源文件移动后素材失效。

### R5: 写入 library.json 必须具备原子性
**断言**：主进程保存 `library.json` 时必须先写入临时文件，再重命名覆盖正式文件；不得直接覆盖正式 JSON。
**示例**：
```ts
await fs.writeFile(tempPath, JSON.stringify(library, null, 2), "utf8");
await fs.rename(tempPath, libraryPath);
```
**违规特征**：`await fs.writeFile(libraryPath, data)` 直接覆盖，且没有临时文件、备份或错误恢复策略。

### R6: 启动路径不得执行全量派生同步
**断言**：应用启动、首屏加载、`useLibraryStore.load()` 和 renderer ready 前后不得执行参数词库、分类词库、标签词库、图片缩略图、提示词解析、AI 分析等全量派生/同步任务；启动阶段只能读取必要的 `library.json`、视图设置和已保存缓存。全量词库重建、提示词扫描、批量图片处理必须由用户触发的数据变更流程执行，或放到可取消、分批让出主线程的后台任务中。
**示例**：
```ts
// 启动：只读取已保存词库，不全量重建。
const [libraryResult, viewSettingsResult] = await Promise.all([
  window.suyanApi.readLibrary(),
  window.suyanApi.readLibraryViewSettings(),
]);
set({ items: libraryResult.data.items, promptLexicons: viewSettingsResult.data.promptLexicons });
```
```ts
// 数据变更后才做同步。
await syncPromptLexiconsFromLibrary(set, get);
```
**违规特征**：启动后立即调用 `syncPromptLexiconsFromLibrary`、扫描全部提示词、合并全部词库、批量生成缩略图或在 renderer 主线程执行长循环，导致窗口已显示但点击、滚动、边栏无响应。

### R7: 性能问题必须保留可定位日志
**断言**：启动、首屏、慢 IPC、菜单切换、批量分析等曾出现卡顿的路径必须保留轻量性能日志，日志不得阻塞 UI，不得写入敏感提示词全文、API Key 或图片内容。新增重任务时必须能通过日志区分是主进程、IPC、renderer 首屏、页面切换还是后台任务耗时。
**示例**：
```ts
window.suyanApi.logStartupEvent("view-switch:committed", {
  durationMs,
  to: view,
});
```
**违规特征**：删除启动/慢操作日志后又新增重同步逻辑；出现“卡住”问题时只能凭主观猜测，无法从日志看到耗时阶段。
