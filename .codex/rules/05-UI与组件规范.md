# UI与组件规范

## 适用范围
适用于 React 组件、Tailwind CSS 样式、设计令牌、页面布局、交互状态和中文界面文案。

## 规则条目

### R1: 首屏必须是提示词卡片浏览界面
**断言**：应用启动后必须直接进入提示词素材浏览工具界面，不得出现营销首页；页面布局必须包含顶部搜索与标签筛选区、中部提示词卡片列表、底部加载进度与“加载更多提示词”操作。卡片必须包含标题、图像、提示词详情、标签、“查看详情”和“复制提示词”操作。
**示例**：
```tsx
export function LibraryView() {
  return (
    <main className="flex h-screen flex-col bg-background text-foreground">
      <PromptLibraryToolbar />
      <PromptCardGrid />
      <PromptLoadProgress />
    </main>
  );
}
```
**违规特征**：启动页是 hero、宣传文案、营销落地页，或首屏缺少搜索、标签筛选、提示词卡片、底部加载进度与加载更多操作。

### R2: 所有颜色必须来自设计令牌
**断言**：项目色值只能定义在 `W:\提示词\src\styles\tokens.css`；组件中只能使用 `bg-background`、`text-foreground`、`border-border`、`bg-panel`、`text-muted`、`bg-primary` 等语义化类名。
**示例**：
```css
@import "tailwindcss";

@theme {
  --color-background: #f7f7f5;
  --color-foreground: #202124;
  --color-panel: #ffffff;
  --color-muted: #6b7280;
  --color-border: #d8d7d2;
  --color-primary: #2563eb;
  --color-primary-foreground: #ffffff;
  --color-danger: #dc2626;
}
```
**违规特征**：组件中出现 `className="bg-blue-500"`、`style={{ color: "#333" }}` 或任意散落硬编码色值。

### R3: 通用组件和业务组件必须分层
**断言**：按钮、输入框、分隔线、空状态等通用组件放入 `W:\提示词\src\components\ui\`；素材列表、图片预览、提示词编辑器等业务组件放入 `W:\提示词\src\features\library\components\`。
**示例**：
```text
W:\提示词\src\components\ui\Button.tsx
W:\提示词\src\components\ui\TextArea.tsx
W:\提示词\src\features\library\components\PromptEditorPane.tsx
```
**违规特征**：`Button.tsx` 内包含素材库保存逻辑，或 `PromptEditorPane.tsx` 被放入全局 `src/components/ui/`。

### R4: 工具操作必须有明确状态反馈
**断言**：导入、保存、删除、导出、复制操作必须在 UI 中显示成功或失败状态；失败信息必须来自结构化错误码映射，不得直接暴露原始异常堆栈。
**示例**：
```ts
export const uiErrorMessage = {
  CLIPBOARD_EMPTY: "剪切板中没有可用图片。",
  IMAGE_COPY_FAILED: "复制图片失败，请重试。",
  LIBRARY_SAVE_FAILED: "保存素材库失败。",
} as const;
```
**违规特征**：`alert(error.stack)`、无提示地失败，或复制/保存按钮点击后没有任何状态变化。

### R5: 文本区域必须适配长提示词
**断言**：提示词编辑区必须使用可滚动多行输入控件；正向提示词和反向提示词不得使用单行 input。
**示例**：
```tsx
<textarea
  className="min-h-48 resize-none rounded-md border border-border bg-panel p-3 text-sm text-foreground outline-none focus:border-primary"
  value={prompt}
  onChange={(event) => setPrompt(event.target.value)}
/>;
```
**违规特征**：用 `<input>` 编辑长 prompt，或文本溢出覆盖保存、复制、删除按钮。

### R6: 边栏菜单必须先响应再挂载重页面
**断言**：点击首页、提示词库、参数词库、分类词库、标签词库、AI 设置、NSFW 设置等边栏菜单时，选中态和轻量反馈必须立即更新；重页面、弹窗、词库工作区、素材管理器等耗时组件必须通过 `startTransition`、下一帧延迟、懒加载或轻量占位挂载，避免点击后 400ms 以上没有反馈。
**示例**：
```tsx
setSidebarActiveView(view);
setPendingMainView(view);
window.setTimeout(() => {
  startTransition(() => {
    setMainView(view);
    setPendingMainView(null);
  });
}, 50);
```
**违规特征**：点击边栏后先同步构建大型页面、菜单树、词库列表或图片列表，导致菜单选中态延迟出现，用户感觉点击无效或软件卡住。

### R7: 词库页面只计算当前视图需要的数据
**断言**：参数词库、分类词库、标签词库必须按当前 `kind` 条件化计算派生数据；打开某一个词库时不得同时构建另外两个词库的树、筛选结果、校验结果、提示词组映射或图像胶囊分组。共享组件应把昂贵 `useMemo`、校验和筛选逻辑按视图拆开或用条件短路。
**示例**：
```tsx
const isParameterWorkspace = kind === "parameters";
const parameterMenuValidation = useMemo(
  () => (isParameterWorkspace ? validatePromptParameterMenuEntries(parameterDrafts) : validValidation),
  [isParameterWorkspace, parameterDrafts],
);
```
**违规特征**：打开标签词库时仍计算参数菜单校验、分类提示词组树；打开分类词库时仍计算全部标签树；一个菜单点击触发多个不相关词库的大量 `filter`、`map`、树构建和分组计算。

### R8: 瀑布流必须顶边对齐分列
**断言**：素材瀑布流展示必须以每一列的顶边为基准从上往下放置图片；首屏可见图片应先从左到右铺满同一顶边，再继续向下分配。不得使用 CSS `column-count` / 多列排版造成“先填满第一列，再进入下一列”的布局，也不得让少量结果全部堆在左侧单列导致右侧大片空白。
**示例**：
```tsx
const columns = distributeItemsByTopEdge(items, columnCount);

return (
  <div className="grid items-start gap-4">
    {columns.map((column) => (
      <div className="grid content-start gap-4">
        {column.map((item) => <PromptImageTile item={item} />)}
      </div>
    ))}
  </div>
);
```
**违规特征**：瀑布流容器使用 `columnCount`、`columns-*`、`break-inside` 作为主要布局方案；2-4 张图片时全部出现在第一列；同一行图片没有以顶部边缘对齐，而是从某张长图底部继续排布。

### R9: 浮层组件必须用 Portal 渲染到 body
**断言**：tooltip、popover、dropdown、menu 等脱离文档流的浮层组件必须通过 `createPortal` 渲染到 `document.body`，并用 `position: fixed` + JS 计算坐标定位。不得依赖 `position: absolute` + 父容器 `z-index`，因为 CSS `overflow`（`hidden`/`auto`/`scroll`）会裁剪所有溢出子元素，无论 z-index 多高。
**示例**：
```tsx
// 正确：Portal + fixed 定位
{isOpen
  ? createPortal(
      <span role="tooltip" style={{ position: "fixed", top, left, zIndex: 9999 }}>
        {label}
      </span>,
      document.body,
    )
  : null}

// 错误：absolute 定位会被父级 overflow 裁剪
<span className="absolute z-50" style={{ top: "100%" }}>{label}</span>
```
**违规特征**：浮层用 `position: absolute` 定位；浮层组件在 `overflow-hidden`/`overflow-y-auto` 容器内不可见；用调整父容器边距（如 `bottom-4` → `bottom-6`）的方式"修复"tooltip 被遮挡——这只是挪动裁剪线，根本问题是 overflow 裁剪。参阅 P029。
