# 技术栈与核心库

## 适用范围
适用于 `package.json`、构建配置、测试配置、依赖安装、技术选型和所有框架 API 使用。

## 规则条目

### R1: 核心技术栈版本必须精确锁定
**断言**：`package.json` 中核心依赖必须使用以下精确版本，不得使用 `latest`、`^`、`~` 或版本范围。
**示例**：
```json
{
  "packageManager": "pnpm@11.9.0",
  "dependencies": {
    "electron": "43.0.0",
    "react": "19.2.7",
    "react-dom": "19.2.7",
    "zustand": "5.0.14",
    "lucide-react": "1.23.0",
    "jszip": "3.10.1",
    "uuid": "14.0.1"
  },
  "devDependencies": {
    "typescript": "6.0.3",
    "vite": "8.1.3",
    "@vitejs/plugin-react": "6.0.3",
    "tailwindcss": "4.3.2",
    "@tailwindcss/vite": "4.3.2",
    "vitest": "4.1.9",
    "electron-builder": "26.15.3",
    "@types/node": "26.1.0",
    "@types/react": "19.2.17",
    "@types/react-dom": "19.2.3"
  }
}
```
**违规特征**：`"react": "^19.2.7"`、`"vite": "latest"`、`"typescript": ">=6"` 或未确认版本的新增框架。

### R2: 包管理器只能使用 pnpm
**断言**：项目根目录必须存在 `pnpm-lock.yaml`，不得提交 `package-lock.json`、`yarn.lock` 或 `bun.lockb`；安装命令必须使用 `pnpm`。
**示例**：
```powershell
pnpm install
pnpm add jszip@3.10.1
pnpm test
```
**违规特征**：出现 `npm install`、`yarn add`、`bun install`，或仓库中同时存在多个包管理器锁文件。

### R3: Vite 只负责渲染层构建
**断言**：`vite.config.ts` 只能配置 React 渲染层、Tailwind 插件和 `@` 路径别名；Electron 主进程与 preload 的构建配置不得混入 React 插件配置。
**示例**：
```ts
import path from "node:path";
import react from "@vitejs/plugin-react";
import tailwindcss from "@tailwindcss/vite";
import { defineConfig } from "vite";

export default defineConfig({
  plugins: [react(), tailwindcss()],
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "src"),
    },
  },
});
```
**违规特征**：在 `vite.config.ts` 中直接配置 `electron/main` 入口，或从 React 组件中导入 `node:fs`。

### R4: 状态管理只能使用 Zustand 或 React 内置状态
**断言**：全局业务状态只能使用 `zustand@5.0.14`；组件局部交互状态优先使用 React `useState`/`useReducer`；不得引入 Redux、MobX、Recoil、Jotai。
**示例**：
```ts
import { create } from "zustand";

type LibraryUiState = {
  selectedItemId: string | null;
  setSelectedItemId: (id: string | null) => void;
};

export const useLibraryUiStore = create<LibraryUiState>((set) => ({
  selectedItemId: null,
  setSelectedItemId: (selectedItemId) => set({ selectedItemId }),
}));
```
**违规特征**：`import { configureStore } from "@reduxjs/toolkit"` 或新增 `mobx`、`recoil`、`jotai` 依赖。

### R5: 单元测试只能使用 Vitest
**断言**：工具函数、数据 schema、IPC 参数序列化、ZIP 导入导出纯函数必须使用 `vitest@4.1.9` 编写单元测试；不得引入 Jest。
**示例**：
```ts
import { describe, expect, it } from "vitest";
import { normalizePromptText } from "@/features/library/utils/normalizePromptText";

describe("normalizePromptText", () => {
  it("trims trailing spaces", () => {
    expect(normalizePromptText("  cat, light  ")).toBe("cat, light");
  });
});
```
**违规特征**：出现 `jest.config.ts`、`import { test } from "@jest/globals"` 或 `jest` 依赖。
